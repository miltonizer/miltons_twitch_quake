ezQuake Manual: 1.8 Changelog

ezQuake 1.8 stable, 27th May 2007

Making QuakeWorld Easier To Setup And Use
  • Supporting 24bit player skins - feature by qqshka, based on the code by jogi
  • Switching video modes on-the-fly in GL - feature by qqshka
  • Switching Fullscreen vs. Windowed Mode on-the-fly in GL - feature by qqshka
  • Client can be associated with demo files to play them directly from Windows - feature by qqshka
  • Equivalents for most of the command-line options feature by qqshka and johnnycz
  • Mouse driven menus - feature by johnnycz, cokeman
  • Better Options menu - feature by johnnycz
  • Ability to set RGB colors on any player skin: r_enemyskincolor, r_teamskincolor, r_skincolormode - feature by qqshka, based on the code by jogi
  • Smooth observing thanks to native TCP QTV connection; qtvplay, qtvlist - feature by qqshka, ported from FTEQW by Spike
  • Use OS keyboard layout - feature by tonik
  • Launching .qtv file with the client established observing connection - feature by qqshka
  • Help text in menu options - feature by johnnycz
  • Better demo browser - feature by johnnycz
  • Creating a demo playlist from zip archive - feature by cokeman
  • Inbuilt teamplay messages (tp_msg*) - feature by Up2nOgOoD[ROCK], johnnycz
  • $colored_armor, $colored_powerups, $colored_short_powerups (displays qrp instead of quad ring pent) - feature by Up2nOgOoD[ROCK], qqshka
  • HUD editor - feature by cokeman, jogi
  • Possibility to use TCL for teamplay scripting - feature by disconnect
  • Ruleset can be changed on-the-fly; you have to be disconnected - feature by disconnect
  • Browsing zip archives in demo browser - feature by cokeman
  • Demo de/compression from within the demo browser; alt+c, alt+d - feature by cokeman
  • Inbuilt qizmo proxy menu navigation support; bind x toggleproxymenu - feature by johnnycz
  • Chatting in unicode support, cyrillic support, charset - feature by tonik
  • Text layer width (conwidth) changing on-the-fly - feature by qqshka
  • Ability to save demos, screenshots and logs not only to Quake dir; cl_mediaroot - feature by qqshka
  • Ability to save configuration into the user's home dir cfg_use_home - feature by qqshka
  • Alpha transparency support for console font; gl_alphafont - feature by qqshka
  • Client loads even if not started from it's directory - feature by qqshka
  • Team overlay table; scr_teaminfo* - feature by qqshka
